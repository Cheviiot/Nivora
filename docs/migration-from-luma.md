# Переход на Nivora

Приложения и их данные удалять не нужно. Nivora устанавливает новые системные
пакеты поверх прежних и не трогает каталоги в домашней директории.

## Почему нужен мигратор

Stapler `v0.1.1` записывает имя исходного репозитория в имя системного пакета:

```text
package+stplr-luma
```

Команда `stplr upgrade` ищет обновление в репозитории с тем же локальным именем. Если
сначала удалить запись `luma`, такой пакет больше не будет найден. `provides` и
`replaces` обрабатываются системным менеджером пакетов после того, как Stapler выбрал
новый рецепт, и не могут исправить этап поиска.

Поэтому простая последовательность `repo rm` → `repo add` → `upgrade` небезопасна: она
оставляет прежние пакеты без канала обновлений.

## Безопасная последовательность

1. Добавьте Nivora, не удаляя старую запись:

   ```bash
   sudo stplr repo add nivora https://github.com/Cheviiot/Nivora.git
   sudo stplr refresh
   ```

2. Установите helper Nivora. Пакетный менеджер заменит прежний helper:

   ```bash
   sudo stplr install nivora/nivora-stplr
   ```

3. Запустите одноразовую миграцию:

   ```bash
   sudo nivora-migrate-from-luma --yes
   ```

4. После успеха проверьте обычное обновление:

   ```bash
   sudo stplr upgrade
   ```

Мигратор:

- обрабатывает только 15 известных package ID;
- останавливается при неизвестном пакете;
- проверяет исчезновение старого и появление нового системного пакета;
- не удаляет старую запись репозитория, пока остался хотя бы один пакет `+stplr-luma`;
- не удаляет пользовательские данные и не выполняет logout.

## Переименованные package ID

| Прежний ID | ID Nivora | Сохранённая команда |
|:--|:--|:--|
| `clash-verge` | `clash-verge-rev` | `clash-verge` |
| `claude-code-desktop` | `claude-desktop` | `claude-desktop`, `claude-code-desktop` |
| `codex-app` | `codex` | `codex-app` |
| `luma-stplr` | `nivora-stplr` | `sl`, `sli`, `slu` и другие aliases |

Desktop-id, systemd units, MIME associations и каталоги данных при этом не переименовываются.

Эта схема проверяется автоматическим DEB/RPM-тестом: прежний системный пакет заменяется
фактическим пакетом Nivora, затем выполняется обновление версии и удаление с контролем
сохранности пользовательского состояния.

## После закрытия старого репозитория

Адрес `https://github.com/Cheviiot/luma` больше не будет работать. Если миграция
остановилась, не удаляйте пакеты и не очищайте их данные. Сохраните вывод команды и
создайте issue в Nivora.
